> **Platform requirement:** This image is built for **x86_64 / amd64** only.

## 4. Start the Stack

```bash

```

This starts 9 services. On first run, the `university-configurator` service:
1. Writes `common_site_config.json` with DB and Redis connection details
2. Detects that the site database does not exist
3. Restores the database from the baked-in backup (~1.3MB)
4. Runs `bench migrate` to apply any pending schema changes
5. Exits successfully, allowing all other services to start

**First-run startup takes 3-5 minutes.** Wait for all services to show healthy:

```bash
docker compose -f docker-compose.prod.yml ps
```

Expected output: all services except `university-configurator` show status `Up` or `Up (healthy)`. The configurator shows `Exited (0)` — this is correct.

## Troubleshooting

### Configurator exits with non-zero code

Check logs: `docker compose -f docker-compose.prod.yml logs university-configurator`

Common causes:
- MariaDB not healthy yet → increase `start_period` in the db healthcheck
- Wrong `MYSQL_ROOT_PASSWORD` → update .env and re-run: `docker compose -f docker-compose.prod.yml up university-configurator`
- Missing or blank `DB_NAME` in .env → add `DB_NAME=_df4394075d9a924d` (the value from .env.example)

### Restore runs on every restart (data keeps getting wiped)

This means `DB_NAME` is blank or unset in your `.env`. The restore script checks whether the database named by `DB_NAME` exists before restoring. With a blank value, the check always returns "not found" and restore runs every time.

Fix: ensure `.env` contains `DB_NAME=_df4394075d9a924d` (or the actual db_name from your site_config.json).

### Frontend loads blank page (no JS/CSS)

The frontend assets may not have been compiled into the image. Verify:
```bash
docker run --rm ghcr.io/{GHCR_USERNAME}/university-erp:v1.0.0 \
  ls /home/frappe/frappe-bench/sites/assets/
```

Expected: frappe/ and erpnext/ directories. If empty, the image was built without `bench build --production` or with sites/assets incorrectly declared as a VOLUME. Rebuild the image from the Containerfile.

If assets are visible in the image but still blank in the browser, run bench build manually inside the running backend:
```bash
docker exec -it university-backend bash -c "cd /home/frappe/frappe-bench && bench build --production"
```

### Port 8080 already in use

Change `HTTP_PUBLISH_PORT` in .env to an available port (e.g., 8888).

### "No module named university_erp"

The image was not built correctly. Rebuild the image from source using the Containerfile (see maintainer build instructions).
