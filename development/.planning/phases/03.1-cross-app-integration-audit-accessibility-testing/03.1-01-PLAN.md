```python
import frappe
from frappe import _

@frappe.whitelist()
def run_checks():
    """Run all cross-app integrity checks. University Admin only."""
    if "University Admin" not in frappe.get_roles():
        frappe.throw(_("Not permitted"), frappe.PermissionError)
    checks = [
        _check_student_enrollment(),
        _check_fees_gl_accounts(),
        _check_employee_faculty_profile(),
        _check_academic_year_validity(),
    ]
    statuses = [c["status"] for c in checks]
    if "fail" in statuses:
        overall = "critical"
    elif "warn" in statuses:
        overall = "degraded"
    else:
        overall = "ok"
    return {"status": overall, "checks": checks}
```

Create `frappe-bench/apps/university_erp/scripts/audit_cross_app_links.py` as a one-time bench console script callable via `bench --site university.local execute university_erp.scripts.audit_cross_app_links.run`.

The script must:

1. **Build doctype-to-app mapping** by scanning all app directories on disk (reuse the same pattern from audit_duplicate_doctypes.py — walk `frappe-bench/apps/*/` for doctype JSONs, extract module, map module to app). Also handle Education-inside-erpnext case.

2. **Discover ALL cross-app Link fields** automatically:
   - For each doctype in each app, iterate its fields
   - If `fieldtype == "Link"` and `options` is a doctype name owned by a DIFFERENT app → record it
   - If `fieldtype == "Dynamic Link"` → record the companion field (the `options` value points to another field on the same doctype that holds the target doctype name)
   - Build a list: `{source_app, source_doctype, field_name, field_type, target_app, target_doctype}`

3. **Test link integrity** for each cross-app link:
   - For Link fields: Run `SELECT COUNT(*) FROM tab{source_doctype} WHERE {field_name} IS NOT NULL AND {field_name} != ''` to get total records with the link populated
   - Then: `SELECT COUNT(*) FROM tab{source_doctype} s WHERE s.{field_name} IS NOT NULL AND s.{field_name} != '' AND NOT EXISTS (SELECT 1 FROM tab{target_doctype} t WHERE t.name = s.{field_name})` to find broken links
   - For Dynamic Link fields: More complex — need to join on the doctype-name field to resolve the target table. Group by the doctype-name field value, then check each group.
   - IMPORTANT: Use `frappe.db.sql()` with parameterized values where possible. For table/column names (which cannot be parameterized in SQL), use `frappe.scrub()` to sanitize and verify against known doctype list.
   - Wrap each check in try/except — some tables may not exist (dead doctypes).

4. **Detect orphan records** (WARNING level): For heavily-referenced doctypes (Student, Employee, Course, Program), check if any records have zero incoming links from expected sources.

5. **Generate CONNECTION_AUDIT.md** with sections:
   - Executive Summary (total cross-app links found, total broken, total warnings)
   - Cross-App Link Inventory table: | Source App | Source Doctype | Field | Type | Target App | Target Doctype | Total Records | Broken Count | Broken % |
   - Broken Link Details (first 10 per type): | Source Doctype | Field | Broken Value | Source Record Name |
   - Severity classification: FAILURE (broken_count > 0), WARNING (orphan records), PASS (all links resolve)
   - Dynamic Link Analysis section: separate table for dynamic links showing the range of target doctypes resolved

6. Write the report to `.planning/phases/03.1-cross-app-integration-audit-accessibility-testing/CONNECTION_AUDIT.md`.
7. Print live results to console as each link type is tested (progress feedback).

Handle edge cases:
- Tables that don't exist in DB (doctype defined on disk but never migrated) — skip with WARNING
- Very large tables (>100k records) — use LIMIT 10000 for broken link sampling, note "sampled" in report
- Dynamic Links where the companion field contains doctype names from the SAME app — skip (not cross-app)
