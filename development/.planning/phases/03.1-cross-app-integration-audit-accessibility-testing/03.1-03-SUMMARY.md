## Decisions Made

- **vitest-axe API:** Used `axe()` function directly since `toHaveNoViolations` matcher is not exported from vitest-axe v1. Violation filtering done manually on results object.
- **Disabled jsdom-unreliable rules:** color-contrast and region rules disabled in axe tests (jsdom cannot compute CSS). Contrast audited separately via Python script against variables.css.
- **Muted text contrast:** Changed `--text-muted` from `--gray-400` (#94A3B8, 2.56:1) to `--gray-500` (#64748B, 4.76:1) which passes WCAG AA.
- **Status color approach:** Added new `--*-text` CSS variables (success-text, warning-text, error-text, info-text) for text-on-white contexts. Kept original vibrant colors for backgrounds and icons where contrast is not an issue.

## Deviations from Plan

None - plan executed exactly as written.

## Issues Encountered

- **vitest-axe import:** `toHaveNoViolations` is not exported from the main module. Resolved by using `axe()` directly and filtering violations manually.
- **AttendanceMarker props:** Required `classInfo` object prop (not individual props). Resolved by providing correct prop structure in test.
- **vitest config scope:** Test include pattern only matched `src/**/*.test.js`. Resolved by adding `tests/**/*.test.js` to vitest.config.js include array.
