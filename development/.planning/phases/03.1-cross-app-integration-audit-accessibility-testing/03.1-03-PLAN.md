**Step 2: Create accessibility test suite** at `portal-vue/tests/accessibility/a11y-audit.test.js`.

The test file must:

a. Import `vitest-axe` matchers and `axe-core`:

b. For EACH shared component (8 components), create a test that:
   - Renders the component with realistic props (use the same prop patterns from existing tests in portal-vue/tests/)
   - Runs `axe(container)` on the rendered HTML
   - Asserts `expect(results).toHaveNoViolations()` for critical/serious only
   - Collects ALL violations (including moderate/minor) into a report array

c. For EACH faculty component (22+ components), create a test that:
   - Renders with minimal required props
   - Runs axe scan
   - Collects violations (same pattern)

IMPORTANT: Some components may need mocked Frappe API calls (useFrappe). Use the existing mock patterns from `portal-vue/tests/` — check how existing tests mock `useFrappe` and follow the same pattern. Components that fetch data on mount will need mocked responses.

**Step 3: Create CSS contrast audit script** at `frappe-bench/apps/university_erp/scripts/audit_accessibility.py`.

This Python script reads CSS files and checks color contrast ratios:
- Parse `variables.css` for all color custom properties
- For each text-color / background-color pair used in `portal.css`, `components.css`, and `university_desk.css`, calculate WCAG contrast ratio
- Contrast ratio formula: `(L1 + 0.05) / (L2 + 0.05)` where L1 is lighter, L2 is darker relative luminance
- WCAG AA requirements: normal text >= 4.5:1, large text (18px+ or 14px+ bold) >= 3:1
- Parse hex colors and CSS var() references (resolve var() to actual values from variables.css)
- Output findings to console

Read ACCESSIBILITY_AUDIT.md from Task 1 to identify all critical and serious violations. Fix EVERY critical and serious issue. Document minor issues as deferred.

**Common fixes to apply (based on typical WCAG violations in Vue apps):**

1. **Color contrast (serious)**: Adjust CSS custom properties in `variables.css` to meet 4.5:1 ratio for normal text, 3:1 for large text. Common fixes:
   - Darken text colors or lighten backgrounds
   - Add explicit contrast-safe color pairs for status indicators (success/warning/error)
   - Ensure dark mode tokens (if any) also meet contrast requirements

2. **Missing form labels (critical)**: In FilterBar.vue and any component with `<select>`, `<input>`:
   - Add `<label>` elements associated via `for`/`id` attributes
   - Or use `aria-label` for visually-hidden labels
   - Ensure every form control has an accessible name

3. **Missing alt text on images/icons (serious)**:
   - Add `alt=""` for decorative images (e.g., chart icons)
   - Add descriptive `alt` text for informational images
   - Add `aria-hidden="true"` for decorative SVG icons
   - Add `role="img"` and `aria-label` for meaningful SVG icons

4. **Missing landmark regions (serious)** in PortalLayout.vue:
   - Ensure `<nav>` for sidebar navigation
   - Ensure `<main>` for content area
   - Add `<aside>` for secondary content if applicable
   - Add skip-to-content link: `<a href="#main-content" class="sr-only focus:not-sr-only">Skip to main content</a>`

5. **Keyboard navigation (critical)**:
   - Ensure all interactive elements in DataTable.vue (sort buttons, pagination) are focusable with `tabindex="0"` where needed
   - Ensure ChartWrapper.vue charts have `role="img"` and `aria-label` describing the chart
   - Ensure modal/dropdown components trap focus

6. **ARIA attributes (serious)**:
   - Add `role="status"` and `aria-live="polite"` to NotificationPanel.vue toast messages
   - Add `aria-sort` attributes to DataTable.vue column headers
   - Add `aria-expanded` to any collapsible sections
   - Add `aria-current="page"` to active navigation item in PortalLayout.vue

7. **Table accessibility (serious)** in DataTable.vue:
   - Ensure `<table>` has `<caption>` or `aria-label`
   - Ensure `<th>` elements have `scope="col"` or `scope="row"`
   - Ensure sort buttons within `<th>` have `aria-label` like "Sort by {column name}"
