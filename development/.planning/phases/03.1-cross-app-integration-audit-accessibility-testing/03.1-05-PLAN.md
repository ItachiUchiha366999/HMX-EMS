  <action>
    Create `audit_all_apis.py` following the established pattern from `audit_faculty_api.py`.

    **Discovery phase:**
    1. Walk the entire `university_erp/university_erp/` directory tree (using `os.walk` from `_get_apps_root()` + `university_erp/university_erp/`).
    2. For each `.py` file, scan for `@frappe.whitelist` decorator using regex: `r'@frappe\.whitelist\(\)?\s*\ndef\s+(\w+)'`.
    3. Build a list of `(module_path, function_name, file_path)` tuples. The module_path should be the Python import path (e.g., `university_erp.university_portals.api.faculty_api`).
    4. Classify each endpoint:
       - **GET endpoints**: function name starts with `get_`, `list_`, `search_`, `check_`, `is_`, `has_`, `count_`, `fetch_`, `calculate_`, `run`
       - **ACTION endpoints**: function name starts with `submit_`, `save_`, `create_`, `update_`, `delete_`, `approve_`, `reject_`, `cancel_`, `send_`, `mark_`, `toggle_`, `set_`
       - **OTHER**: anything else
    5. Group by module (e.g., "Faculty API", "Portal API", "Analytics API", "Student API v1", etc.).

    **Testing phase:**
    6. Set user context to Administrator: `frappe.set_user("Administrator")`.
    7. For each GET endpoint, attempt to call it with no arguments (wrapped in try/except).
       - PASS: returns a value without unhandled exception
       - FAIL: raises an exception (capture error type and first 200 chars of message)
       - SKIP: endpoint requires specific arguments that cannot be guessed (document as "needs args")
    8. For ACTION endpoints, do NOT call them (they modify data). Just verify the function is importable: `IMPORTABLE` or `IMPORT_FAIL`.
    9. Track total: endpoints discovered, GET tested, GET passed, GET failed, ACTION importable, IMPORT failures.

    **Output:**
    10. Generate `API_COMPREHENSIVE_AUDIT.md` in the phase directory with:
        - Executive summary (total endpoints, pass/fail counts)
        - Table per module group: | # | Endpoint | Type | Status | Error |
        - Section for import failures (these indicate broken code)
        - Section for endpoints needing specific arguments

    11. `def run():` entry point.

    **Execute:**

    ```bash
    bench --site university.local execute university_erp.scripts.audit_all_apis.run
    ```
