# Phase 2 -- UI Design Contract

> Visual and interaction contract for the Shared Component Library. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom design system) |
| Preset | not applicable |
| Component library | Custom Vue 3 SFC components using existing CSS variable tokens |
| Icon library | Material Symbols Outlined (already used in Sidebar.vue) |
| Font | Inter, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif (from `--font-family` in variables.css) |

**Note:** PortalLayout.vue currently declares `'Lexend'` as primary font in its scoped style. Phase 2 components MUST use the `var(--font-family)` token (Inter) from variables.css. The Lexend reference in PortalLayout.vue should be corrected to `var(--font-family)` during Phase 2 implementation to unify typography.

## Spacing Scale

Declared values from `variables.css` (all multiples of 4):

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, inline padding, trend arrow margin |
| sm | `--space-2` | 8px | Compact element spacing, checkbox gap |
| md | `--space-4` | 16px | Default element spacing, card inner gap |
| lg | `--space-6` | 24px | Card padding, section padding |
| xl | `--space-8` | 32px | Layout gaps between card groups |
| 2xl | `--space-12` | 48px | Major section breaks, empty state padding |
| 3xl | `--space-16` | 64px | Page-level vertical spacing |

**Additional tokens used sparingly:**
- `--space-3` (12px) -- table cell padding, filter input padding
- `--space-5` (20px) -- stat-card padding, form group margin

Exceptions: Touch targets on mobile use `min-height: 44px` per existing portal.css pattern (line 957).

## Color

All colors reference existing CSS variables from `variables.css`.

### Light Mode (default)

| Role | CSS Variable | Hex Value | Usage |
|------|-------------|-----------|-------|
| Dominant (60%) | `--gray-50` | #F8FAFC | Page background, main content area |
| Secondary (30%) | `--white` | #FFFFFF | Cards, table wrapper, sidebar, filter bar, chart container |
| Accent (10%) | `--primary` | #4F46E5 | See reserved-for list below |
| Destructive | `--error` | #EF4444 | Destructive actions only |

### Dark Mode

Dark mode is implemented via `[data-theme="dark"]` selector on `<html>`. Components use CSS variables exclusively -- no hardcoded hex values.

| Token | Light Value | Dark Value |
|-------|-------------|------------|
| `--bg-surface` | `--gray-50` (#F8FAFC) | `--gray-900` (#0F172A) |
| `--bg-card` | `--white` (#FFFFFF) | `--gray-800` (#1E293B) |
| `--text-primary` | `--gray-900` (#0F172A) | `--gray-50` (#F8FAFC) |
| `--text-secondary` | `--gray-500` (#64748B) | `--gray-400` (#94A3B8) |
| `--border-default` | `--gray-200` (#E2E8F0) | `--gray-700` (#334155) |
| `--border-subtle` | `--gray-100` (#F1F5F9) | `--gray-800` (#1E293B) |

**Implementation:** Add a `[data-theme="dark"]` block to variables.css that reassigns the semantic tokens above. Components reference only the semantic tokens, never raw `--gray-*` values for surfaces, text, or borders.

### Accent reserved for:

- Active sidebar navigation item background (`--primary-light`) and text (`--primary`)
- Active tab indicator border/background
- Primary CTA buttons (`btn-primary` uses `--primary`)
- KPI card icon background tint (e.g., `--primary-light` for neutral KPIs)
- Chart series default first color
- Focus ring on form inputs (`box-shadow: 0 0 0 3px var(--primary-light)`)
- Filter bar "Apply Filters" button

### Status Colors (KPI card and badge semantic colors)

| Status | Color Variable | Hex | Light BG Variable | Icon |
|--------|---------------|-----|-------------------|------|
| Good / On Target | `--success` | #10B981 | `--success-light` (#D1FAE5) | `trending_up` or `check_circle` |
| Warning / Below Target | `--warning` | #F59E0B | `--warning-light` (#FEF3C7) | `trending_flat` or `warning` |
| Critical / Off Target | `--error` | #EF4444 | `--error-light` (#FEE2E2) | `trending_down` or `error` |
| Informational | `--info` | #3B82F6 | `--info-light` (#DBEAFE) | `info` |

KPI cards apply a 3px left border in the status color plus the light background tint on the status icon area, matching the existing `.stat-icon.success` / `.stat-icon.warning` / `.stat-icon.error` pattern from components.css.

### Chart Series Palette

| Series | Color | Usage |
|--------|-------|-------|
| 1 | #4F46E5 (primary) | Default first series |
| 2 | #10B981 (success) | Second series / comparison |
| 3 | #F59E0B (warning) | Third series |
| 4 | #3B82F6 (info) | Fourth series |

Heatmap gradient: `--success-light` (#D1FAE5) at low end through `--success` (#10B981) at mid through `--success-dark` (#065F46) at high end.

## Dashboard Composition

**Focal point:** The KPI card row is the primary visual anchor of any dashboard view. It occupies the top of the content area, above charts and tables, providing at-a-glance status before the user scrolls to detail components.

**Visual hierarchy (top to bottom):**
1. KPI card row (4-column grid at desktop) -- primary focal point
2. Chart row (1-2 charts side-by-side at desktop)
3. Data table (full width, with filter bar above or inline)
4. Notification panel (sidebar or below-fold section)

## Component Visual Contracts

### COMP-01: KPI Counter Card

```
+-------------------------------------------------------+
|  [icon-bg]  Label text (14px, normal, gray-500)        |
|  [  icon ]  VALUE (24px, semibold, gray-900)           |
|             trend_arrow +12.5% vs last sem (12px)      |
+-------------------------------------------------------+
   ^--- 3px left border in status color
```

- **Container:** `--white` background, `--radius-xl` (12px) border-radius, `--border-width` solid `--gray-200` border, 3px left border in status color, `--space-5` (20px) padding
- **Icon area:** 48x48px, `--radius-lg` (8px), tinted background (`--success-light`, `--warning-light`, `--error-light`, or `--primary-light`)
- **Trend indicator:** Material Symbols icon (`trending_up` / `trending_down` / `trending_flat`) + percentage text. Green for positive, red for negative, gray for flat.
- **Hover:** `box-shadow: var(--shadow-md)` (matches existing `.card:hover`)
- **Loading state:** Skeleton placeholder matching card dimensions using existing `.skeleton` animation from components.css
- **Dark mode:** Card background swaps to `--bg-card`, text to `--text-primary`, border to `--border-default`

### COMP-02: Chart Wrapper

- **Container:** Card wrapper (`--white` bg, `--radius-xl`, `--gray-200` border)
- **Header:** `--space-4` padding, chart title at `--text-base` (16px) / `--font-semibold`, optional subtitle at `--text-xs` / `--gray-500`
- **Chart area:** Minimum height 240px, `--space-5` padding
- **ApexCharts theme:** Pass `fontFamily: var(--font-family)`, colors from chart series palette above, tooltip background `--white` with `--shadow-lg`
- **Loading state:** Skeleton rectangle matching chart dimensions, animated with existing `.skeleton` keyframes
- **Error state:** Empty chart area with centered icon (`error_outline`, 48px, `--gray-300`) + message "Unable to load chart data" (`--text-sm`, `--gray-500`) + "Retry" ghost button
- **Empty state:** Centered icon (`bar_chart`, 48px, `--gray-300`) + "No data available for this period" (`--text-sm`, `--gray-500`)
- **Drill-down cursor:** `cursor: pointer` on clickable data points; chart emits `drill-down` event

### COMP-03: Data Table

- **Wrapper:** `--white` bg, `--radius-xl`, `--gray-200` border, `overflow-x: auto`
- **Header bar:** `--space-3` y / `--space-4` x padding, contains: search input (left), row count text (center-left), export buttons (right)
- **Column headers:** `--text-xs`, `--font-semibold`, `--gray-500`, uppercase, `--gray-50` background, `--space-3` y / `--space-4` x padding. Sortable columns show `arrow_upward` / `arrow_downward` icon (12px) on hover; active sort icon in `--primary`
- **Body cells:** `--text-sm`, `--gray-700`, `--space-3` y / `--space-4` x padding, `--gray-100` bottom border
- **Row hover:** `--gray-50` background
- **Row selection:** 16x16px checkbox with `accent-color: var(--primary)`, aligned in first column. Selected row gets `--primary-light` background tint.
- **Pagination:** Below table, `--space-3` padding, flex layout: "Showing 1-20 of 245" text (left, `--text-sm`, `--gray-500`), page buttons (right) using `btn-secondary` / `btn-primary` for current page
- **Search input:** Inside table header bar, `--gray-50` background, `--gray-200` border, `--radius-lg`, search icon (`search`, `--gray-400`) prefix, `--text-sm` input
- **Loading state:** Table skeleton -- header row + 5 body rows of skeleton rectangles
- **Empty state:** Full-width cell spanning all columns, centered: `table_chart` icon (48px, `--gray-300`) + "No records found" heading (`--text-base`, `--font-semibold`) + "Try adjusting your filters" body (`--text-sm`, `--gray-500`)
- **Export buttons:** Icon-only buttons (`btn-icon btn-secondary btn-sm`), `picture_as_pdf` for PDF (`aria-label="Export to PDF"`), `table_view` for Excel (`aria-label="Export to Excel"`), with tooltip via `data-tooltip`

### COMP-04: Filter Bar

- **Container:** `--white` bg, `--radius-xl`, `--gray-200` border, `--space-4` padding, flex row with wrap, `--space-3` gap between filter items
- **Filter inputs:** Use existing `.form-input` / `.form-select` styles from components.css at `--text-sm`, `--radius-lg`
- **Filter labels:** `--text-xs`, `--font-normal`, `--gray-500`, `--space-1` margin-bottom
- **Date range:** Two date inputs with "to" text separator (`--text-sm`, `--gray-400`)
- **Select dropdowns:** Native `<select>` styled with `.form-select`, populated from Frappe API
- **Action area:** "Apply Filters" button (`btn-primary btn-sm`) + "Clear Filters" button (`btn-ghost btn-sm`)
- **Collapsed state on mobile:** Filters collapse behind a "Filters" toggle button; expanded state shows vertically stacked

### COMP-05 + COMP-06: Export (PDF + Excel)

- **Trigger:** Icon buttons in DataTable header bar (see COMP-03 export buttons)
- **PDF (jsPDF):** Exports current page data as a formatted table. Header row uses `--primary` background with white text. Body alternates white and `--gray-50` rows. Page title at top in `--text-base` (16px) / `--font-semibold`. Timestamp footer in `--text-xs`.
- **Excel (ExcelJS):** Exports current page data. Header row semibold with `--primary` hex (#4F46E5) fill. Auto-width columns. Sheet name matches the view title.
- **Feedback:** Brief toast notification "Exported to PDF" / "Exported to Excel" in `--success-light` bg with `--success-dark` text, auto-dismiss after 3 seconds, bottom-right position

### COMP-07: Report Viewer

- **Container:** Embedded inline within portal module page, no standalone page
- **Filter section:** Dynamic filters rendered from Frappe report definition. Uses same `.form-input` / `.form-select` / `.form-label` styles as COMP-04. "Run Report" button (`btn-primary btn-sm`).
- **Results:** Rendered inside a COMP-03 DataTable instance, inheriting all table styles, pagination, and export
- **Loading:** Full skeleton table while report executes
- **Error:** Alert banner (`alert-error` from components.css) above the empty table: "Report failed to load. Check your filters and try again."
- **Empty result:** DataTable empty state (see COMP-03)

### COMP-08: Notification/Alert Panel

- **Container:** Card wrapper (`--white` bg, `--radius-xl`, `--gray-200` border)
- **Header:** "Notifications" title (`--text-base` 16px, `--font-semibold`), "Mark all read" ghost button (right)
- **Notification item:** Uses existing `.notification-item` pattern from portal.css (theme). 40x40px icon area with category-specific tint (assignment=primary, grade=warning, deadline=error, schedule=info). Title (`--text-sm`, `--font-semibold`), body (`--text-sm`, `--gray-500`), timestamp (`--text-xs`, `--gray-400`).
- **Unread indicator:** 8px circle dot in `--primary` color, positioned left of item or as left border
- **Read state:** No dot, slightly reduced opacity (0.7) on timestamp
- **Dismiss:** Swipe-to-dismiss on mobile, "x" icon button on hover for desktop
- **Empty state:** Bell icon (`notifications_none`, 48px, `--gray-300`) + "All caught up" heading + "No new notifications" body
- **Max visible:** Show 5 most recent, "View all notifications" link at bottom (`--text-sm`, `--primary`)

## Interaction Contracts

### Loading States
All components use skeleton loading (existing `.skeleton` animation from components.css). No spinners.

| Component | Skeleton Shape |
|-----------|---------------|
| KPI Card | Rectangle matching card dimensions (120px height) |
| Chart | Rectangle matching chart area (240px height) |
| Table | Header row + 5 body rows of horizontal bars |
| Filter Bar | 3-4 horizontal bars at input height |
| Notifications | 3 list items with icon circle + text bars |

### Transitions
- Card hover shadow: `var(--transition-base)` (200ms) with `var(--ease-in-out)`
- Tab switch: `var(--transition-fast)` (150ms)
- Skeleton animation: 1.5s infinite (existing keyframe)
- Toast notification: Slide in from bottom-right over 200ms, auto-dismiss after 3000ms

### Responsive Breakpoints

| Breakpoint | Layout Change |
|------------|---------------|
| >= 1024px | KPI cards in 4-column grid, tables full width, charts side-by-side possible |
| 768px - 1023px | KPI cards in 2-column grid, tables full width with horizontal scroll, charts stack |
| < 768px | KPI cards single column, filter bar collapses behind toggle, sidebar hidden |

### Accessibility
- All interactive elements have visible focus rings (`box-shadow: 0 0 0 3px var(--primary-light)`)
- Table sort columns have `aria-sort` attribute
- Checkboxes have associated `<label>` elements
- Chart has `role="img"` with `aria-label` describing the data summary
- Color is never the sole indicator -- status always includes icon + text alongside color
- Toast notifications use `role="status"` with `aria-live="polite"`
- Icon-only export buttons have explicit `aria-label`: "Export to PDF" and "Export to Excel"

## Dark Mode Contract

Toggle mechanism: Set `data-theme="dark"` attribute on `<html>` element. All components read CSS variables only.

New semantic tokens to add to `variables.css`:

```css
:root {
  --bg-surface: var(--gray-50);
  --bg-card: var(--white);
  --text-primary: var(--gray-900);
  --text-secondary: var(--gray-500);
  --text-muted: var(--gray-400);
  --border-default: var(--gray-200);
  --border-subtle: var(--gray-100);
  --skeleton-base: var(--gray-200);
  --skeleton-shine: var(--gray-100);
}

[data-theme="dark"] {
  --bg-surface: var(--gray-900);
  --bg-card: var(--gray-800);
  --text-primary: var(--gray-50);
  --text-secondary: var(--gray-400);
  --text-muted: var(--gray-500);
  --border-default: var(--gray-700);
  --border-subtle: var(--gray-800);
  --skeleton-base: var(--gray-700);
  --skeleton-shine: var(--gray-600);
}
```

All Phase 2 components MUST reference these semantic tokens for surfaces, text, and borders. Raw `--gray-*` tokens may only be used for status light backgrounds (e.g., `--success-light`) which remain consistent across themes.
