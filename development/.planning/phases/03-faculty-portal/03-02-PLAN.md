    **1. Implement backend endpoints in faculty_api.py** (replace stub implementations from Plan 01):

    a) `get_students_for_grading(course, assessment_plan)`:
       - Verify faculty teaches this course via Teaching Assignment
       - Get enrolled students for this course from Course Enrollment
       - Get existing Assessment Result records for each student + assessment_plan
       - Return: list of {student, student_name, roll_no, grades: [{assessment_name, marks, grade, docstatus (0=draft, 1=submitted)}]}
       - Also return: {assessments: [{name, assessment_type, max_marks}]} from Assessment Plan

    b) `save_draft_grade(student, course, assessment_plan, assessment_name, marks)`:
       - Verify faculty authorization
       - Find or create Assessment Result (docstatus=0, draft) for this student + course + assessment_plan
       - Update the specific assessment criteria row with the marks value
       - Let UniversityAssessmentResult.validate() compute the grade automatically
       - Return: {grade: computed_grade, marks: entered_marks, status: "saved"}

    c) `submit_grades(course, assessment_plan)`:
       - Verify faculty authorization
       - Find all draft Assessment Results for this course + assessment_plan
       - Submit each (docstatus = 1) via doc.submit()
       - Return: {submitted_count, message}

    d) `get_grade_analytics(course, assessment_plan)`:
       - Return: {distribution: {A: 5, B+: 8, B: 12, ...}, average: 72.3, pass_count: 48, fail_count: 12, at_risk_count: 5, at_risk_threshold: 40}
       - at_risk = students with average marks below 40% of max marks

    e) `get_student_list(course, section=None, search=None, start=0, page_length=20, order_by="student_name")`:
       - Verify faculty teaches this course
       - Return paginated list: {data: [{student, student_name, roll_no, enrollment_no, image, attendance_percentage, current_grade}], total_count}
       - attendance_percentage calculated server-side from Student Attendance records
       - current_grade from latest Assessment Result

    f) `get_student_detail(student, course)`:

    **2. Create FacultyGradeGrid.vue** at `portal-vue/src/components/faculty/FacultyGradeGrid.vue`:
    This is a CUSTOM component -- NOT DataTable. Spreadsheet-like editable grid.

    - Props: `course` (string), `assessmentPlan` (string)
    - Fetches data on mount via `useFacultyApi().getStudentsForGrading(course, assessmentPlan)`
    - Grid container: `--bg-card`, `--radius-xl`, `--border-default` border, `overflow-x: auto`
    - Column headers: assessment names from API response, `--text-xs`, `--font-semibold`, `--gray-500`, `--gray-50` bg, uppercase
    - Student name column: frozen (sticky left), `--bg-card` bg, 180px min-width, `--text-sm`, `--font-semibold`
    - Editable cells: `<input type="text">` elements, `min-width: 80px`, `--space-2` padding, `--text-sm`, `--font-normal`, `--border-subtle` border
    - Focus state: `--primary` border 2px, `box-shadow: 0 0 0 3px var(--primary-tint)`
    - Keyboard navigation:
      - Tab: move to next cell in same row (prevent default tab behavior)
      - Enter: move to same column in next row
      - Use a ref-based cell registry: `cellRefs[rowIndex][colIndex]` to manage focus
    - Auto-save with debounce:
      - Track cell states in a reactive Map: `cellStates = shallowReactive(new Map())` -- keys are `${studentName}-${assessmentName}`, values are `'idle'|'saving'|'saved'|'error'`
      - On input: set state to 'saving', call debounced (500ms) `saveDraftGrade({student, course, assessment_plan, assessment_name, marks: value})`
      - On success: set state to 'saved', after 1500ms set back to 'idle'
      - On error: set state to 'error'
      - Visual: green checkmark icon (`check`, 12px, `--success`) that fades in (200ms) when 'saved', fades out after 1500ms. Use `role="status"` with `aria-live="polite"` and sr-only text "Saved".
    - Use `shallowRef` for the grade matrix data to avoid deep reactivity overhead (per Research pitfall 3).
    - Computed grade column: read-only, `--gray-50` bg, `--text-sm`, `--font-semibold` -- shows grade letter from API response
    - At-risk row: `--error-tint` bg, `--error` left border 3px. Threshold: below 40% of max marks.
    - Status badge: "Draft" (`--warning-light` bg, `--warning-dark` text) or "Submitted" (`--success-light` bg, `--success-dark` text), `--radius-full`, `--text-xs`
    - Submit bar: `--bg-card`, `--border-default` top border. "Submit Grades" button (`btn-primary`). Auto-save timestamp: `--text-xs`, `--gray-400`, "Auto-saved {time}".
    - On submit: call `submitGrades(course, assessmentPlan)`. On success: useToast "Grades submitted successfully". All cells become read-only, `--gray-100` bg. Show "Grades have been submitted and are now locked for editing." banner.
    - Empty state: "No courses to grade" / "You have no active courses requiring grade entry this semester."
    - Loading: SkeletonLoader (header + 8 rows of cell placeholders)
    - ARIA: `role="grid"` on table, `role="row"` on rows, `role="columnheader"` on headers, `role="gridcell"` on cells with `aria-label="{assessment} for {student}"`
    - Course/assessment selector: dropdown above grid to select course and assessment plan from faculty's courses. On change, refetch data.
