# Phase 3 -- UI Design Contract

> Visual and interaction contract for the Faculty Portal. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom design system, inherited from Phase 2) |
| Preset | not applicable |
| Component library | Custom Vue 3 SFC components + Phase 2 shared components (KpiCard, ChartWrapper, DataTable, FilterBar, NotificationPanel, ReportViewer, SkeletonLoader, ToastNotification) |
| Icon library | Material Symbols Outlined (established in Phase 1) |
| Font | Inter, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif (`var(--font-family)` from variables.css) |

**Inherited from Phase 2:** All design tokens, color system, dark mode contract, spacing scale, typography scale, and shared component visual contracts carry forward unchanged. This spec defines only the faculty-portal-specific visual and interaction contracts.

## Spacing Scale

Inherited from Phase 2 UI-SPEC. All values from `variables.css`:

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, inline padding, trend arrow margin |
| sm | `--space-2` | 8px | Compact element spacing, checkbox gap, grid cell padding |
| md | `--space-4` | 16px | Default element spacing, tab content padding |
| lg | `--space-6` | 24px | Card padding, section padding |
| xl | `--space-8` | 32px | Layout gaps between card groups |
| 2xl | `--space-12` | 48px | Major section breaks, empty state padding |
| 3xl | `--space-16` | 64px | Page-level vertical spacing |

**Additional tokens used sparingly:**
- `--space-3` (12px) -- table cell padding, grade grid cell padding, filter input padding
- `--space-5` (20px) -- stat-card padding, form group margin, timetable slot padding
- `--space-10` (40px) -- form section separators

Exceptions: Touch targets on mobile use `min-height: 44px` per portal.css pattern. Grade grid cells use `min-width: 80px` for numeric input.

## Page Layout & Navigation

### Sidebar Navigation (updates to navigation.js)

The tab-based approach reduces sidebar items. Faculty gets 3-4 top-level sidebar entries:

| Sidebar Item | Icon | Path | Contains (tabs) |
|-------------|------|------|-----------------|
| Dashboard | `dashboard` | `/faculty` | Pending Tasks + Overview |
| Teaching | `school` | `/faculty/teaching` | Timetable, Attendance, Grades, Students |
| My Work | `work` | `/faculty/work` | Leave, Workload, Research, OBE |
| Notices | `campaign` | `/faculty/notices` | Announcements archive with filters |

### Tab Component Contract

```
+--[ Tab 1 ]--[ Tab 2 ]--[ Tab 3 ]--[ Tab 4 ]-------+
|  ================================================== |
|  (2px bottom border in --primary for active tab)     |
+-----------------------------------------------------+
|                                                       |
|  Tab content area (--space-4 top padding)            |
|                                                       |
+-----------------------------------------------------+
```

- **Tab bar:** `--bg-card` background, `--border-default` bottom border (1px), `--space-4` horizontal padding
- **Tab item:** `--space-3` x-padding, `--space-2` y-padding, `--text-sm`, cursor pointer
- **Active tab:** `--font-semibold`, `--primary` text color, 2px bottom border in `--primary`
- **Inactive tab:** `--font-normal`, `--text-secondary` color, no bottom border
- **Hover (inactive):** `--text-primary` color, `--gray-100` background
- **Transition:** Tab switch uses `var(--transition-fast)` (150ms)
- **Content area:** `--space-4` top padding below tab bar

- **KPI cards row:** 4-column grid at desktop (reuse Phase 2 KpiCard). Each card is clickable -- navigates to the relevant tab.

  - "Unmarked Attendance" -- `--warning` status color (if >0) / `--success` (if 0), `fact_check` icon

  - Already marked: "Marked" text in `--success` with `check_circle` icon, row background `--gray-50`, text opacity 0.7
- **Overview tab:** Charts -- weekly attendance trend (ChartWrapper line), grade distribution summary (ChartWrapper bar), recent announcements (NotificationPanel, max 3 items)

- **Computed grade column:** Read-only, `--gray-50` background, `--text-sm`, `--font-semibold`
- **Side panel (analytics):** 320px width at desktop (flex layout), stacks below grid on tablet/mobile. Contains ChartWrapper (bar chart -- grade distribution) + KPI text items (class average, pass/fail count, at-risk count). Updates live as grades are entered.
- **Submit bar:** `--bg-card`, `--border-default` top border, "Submit Grades" button (`btn-primary`), auto-save timestamp (`--text-xs`, `--gray-400`, "Auto-saved {time}")
- **Submitted/locked state:** All cells read-only (no focus ring), `--gray-100` bg, "Grades Submitted" banner

### Leave Management

**Faculty's own leave (My Work tab > Leave sub-tab):**

```
+---------------------------------------------------+
| Leave Balance                                       |
| +----------+ +----------+ +----------+             |
| | Casual   | | Medical  | | Earned   |             |
| | 8 / 12   | | 3 / 10  | | 15 / 20  |             |

|                                                     |

| +---+--------+---------+--------+--------+         |
| |   | Type   | Dates   | Reason | Status |         |

+---------------------------------------------------+
```

- **Leave balance cards:** 3-column grid, each card uses KpiCard pattern. Value = "{used} / {total}", label = leave type name. Progress bar below value: `--success` fill if < 75% used, `--warning` if 75-90%, `--error` if > 90%.
  - Progress bar: 4px height, `--gray-200` track, fill color per threshold, `--radius-full`
- **Apply form:** Modal or slide-over panel (320px from right). Fields: Leave Type (select), From Date, To Date, Reason (textarea). "Submit Leave Request" button (`btn-primary`). "Cancel" button (`btn-ghost`).
- **Recent applications table:** DataTable with columns: Type, From Date, To Date, Reason (truncated), Status (badge). Status badges use semantic colors (Approved=success, Rejected=error, Pending=warning).

**Student leave approval queue (Teaching tab > Students sub-tab or dedicated view):**

- DataTable with columns: Student Name, Roll No, From Date, To Date, Reason, Actions
- Actions column: "Approve" button (`btn-sm`, `--success` bg, white text) + "Reject" button (`btn-sm`, `btn-ghost`, `--error` text)
- Bulk approve: Multi-select checkboxes + "Approve Selected" button above table (`btn-primary btn-sm`)
- Reject requires reason: Clicking "Reject" opens inline text input below the row for rejection reason + "Confirm Reject" button (`btn-sm`, `--error` bg)

### Accessibility

Inherited from Phase 2 plus:
- Grade grid cells have `role="gridcell"` and `aria-label="{assessment} for {student}"`

## Dark Mode Contract

All Phase 3 components use semantic tokens from variables.css. No hardcoded hex values. Dark mode works automatically via `[data-theme="dark"]`.

Phase 3 additions to semantic token usage:

| Element | Light Token | Dark Behavior |
|---------|-------------|---------------|
| Grade cell editable bg | `--bg-card` (#FFFFFF) | `--bg-card` (#1E293B) |
| Grade cell focus ring | `--primary-light` | Same (semi-transparent works in both) |
| Absent row highlight | `--error-light` (#FEE2E2) | Needs dark equivalent: `rgba(239, 68, 68, 0.15)` |
| At-risk row highlight | `--error-light` (#FEE2E2) | Same dark equivalent: `rgba(239, 68, 68, 0.15)` |
| Timetable occupied slot | `--primary-light` (#EEF2FF) | `rgba(79, 70, 229, 0.15)` |
| Already-marked attendance | `--gray-100` | `--border-subtle` (--gray-800) |

**Implementation:** For light background tints that need dark mode equivalents, use `rgba()` with the accent color at 15% opacity. Add these as CSS variables in variables.css:

```css
:root {
  --error-tint: var(--error-light);
  --primary-tint: var(--primary-light);
}

[data-theme="dark"] {
  --error-tint: rgba(239, 68, 68, 0.15);
  --primary-tint: rgba(79, 70, 229, 0.15);
}
```

Phase 3 components use `--error-tint` and `--primary-tint` for row/cell highlights instead of raw `--error-light` / `--primary-light`.
