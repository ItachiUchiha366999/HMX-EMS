    **1. Implement backend endpoints in faculty_api.py** (replace stubs from Plan 01):

    a) `get_leave_balance()`:
       - Get current faculty employee
       - Use `frappe.db.sql` to query Leave Allocation for this employee in current leave period
       - For each leave type: get total_leaves_allocated, leaves_used (from Leave Application with status=Approved + docstatus=1)
       - Return: [{leave_type, total, used, balance}]

    b) `apply_leave(leave_type, from_date, to_date, reason)`:
       - Create Leave Application doc: employee=current_faculty.name, leave_type, from_date, to_date, description=reason, status="Open"
       - doc.insert() and doc.submit() if auto-submit is appropriate, else just insert
       - Return: {name, status, message: "Leave request submitted"}

    c) `get_leave_history(start=0, page_length=10)`:
       - Get Leave Applications for current faculty employee
       - Return: {data: [{name, leave_type, from_date, to_date, description, status, posting_date}], total_count}

    d) `get_student_leave_requests(start=0, page_length=20)`:
       - Find student leave applications for courses taught by current faculty
       - Check for "Student Leave Application" doctype first. If it does not exist, check for Leave Application records with custom student applicant fields or Student Attendance with leave status.
       - Return: {data: [{name, student_name, roll_no, from_date, to_date, reason, status}], total_count}

    e) `approve_student_leave(name)`:
       - Set status to "Approved", approved_by = current faculty
       - Return: {status: "Approved", message: "Leave request approved"}

    f) `reject_student_leave(name, reason)`:
       - Set status to "Rejected", rejection_reason = reason
       - Return: {status: "Rejected", message: "Leave request rejected"}

    g) `get_lms_courses()`:
       - Check if LMS doctypes exist: `frappe.db.exists("DocType", "LMS Course")`
       - If not: return {available: false, courses: []}
       - Get LMS Courses where instructor = current faculty
       - For each: count lessons, assignments, quizzes
       - Return: {available: true, courses: [{name, title, lesson_count, assignment_count, quiz_count}]}

    h) `get_lms_course_content(course)`:
       - Return: {lessons: [{name, title, content, idx}], assignments: [{name, title, due_date, status}], quizzes: [{name, title, question_count, status}]}

    i) `save_lms_content(doctype, name=None, data=None)`:
       - If name: update existing doc
       - Else: create new doc
       - Return: {name, message: "Content saved"}

    j) `delete_lms_content(doctype, name)`:
       - Delete the doc
       - Return: {message: "Content deleted"}

    k) `get_publications()`:
       - Get Faculty Publication records linked to current faculty
       - Group by publication_type
       - Return: {total: N, by_type: {Journal: 6, Conference: 4, Book: 2}, publications: [{name, title, publication_type, journal_name, year, authors, citations, abstract, doi}]}

    l) `get_copo_matrix(course)`:
       - Use COPOAttainmentCalculator or get_copo_matrix() from accreditation module
       - Return: {course, cos: ["CO1","CO2",...], pos: ["PO1","PO2",...], matrix: [[2.8, 1.5, ...], [...]]}

    m) `get_copo_student_detail(course, co)`:
       - Return student-level attainment for a specific CO
       - Return: {co, students: [{student, student_name, attainment_value}]}

    n) `get_workload_summary()`:
       - Run faculty_workload_summary report for current faculty
       - Also get department average by running for all faculty in same department
       - Return: {personal: {hours_per_week, total_courses, total_credits, total_students}, department_avg: {hours_per_week, total_courses, total_credits, total_students}, heatmap: [{day, timeslot, occupied: bool, course_name}]}
